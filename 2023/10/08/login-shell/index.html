LOGIN SHELL VS NON-LOGIN SHELL

I decide to change to wayland, and with the terminal of foot configuration, i get confused about login-shell vs non-login shell.

By reading this page, i think i may get clear to it.

With login-shell, it will automatically read all configuration files end with profile. For example of bash, including /etc/profile and ~/.bash_profile.

With non-login shell, it will automatically read all configuration files of login-shell, but also reading rc files. For example of bash, it will read /etc/bashrc and ~/.bashrc.

So admin can use login shell, but users which have their own configurations better to use non-login shell.
