PASSWORD MANAGER AND GNUPG

pass is a command line password manager which i will use after watching Luck Smith’s video, cause it can use two-factor, and Github recently needs to use. Pass uses gpg to encrypt password, so we will talk both in this article.

GnuPG

By default gpg will store keys in ~/.gnugp, it is absolutely not i want, so we set $GNUPGHOME variable to change.

And we also need to set the correct permission according to archwiki.

Files under $GNUPGHOME need the permission 600, and it will auto set after we generate gpg key.

Note:

The user id will shows in the output of generation, and we can also check with gpg --list-keys.

passmenu

Passmenu is a tool(script) to help us to get our password with dmenu/dmenu-dl(default in wayland), nice and easy, automatically copy password to clipboard. But in my case, i use bemenu in wayland. So i need copy the script and modify something. After that, we can give it a keybinding to launch passmenu and choose our password.

pass-otp

With pass-otp, we can generate two-factor code which will be used in github. And without a cellphone to scan the QR image or install a cellphone app to generate the six digital code.

At first we need to install pass-otp.

In order to scan QR image to QR code, we need zbarimg.

After installed that, we can got to our github setting(password and authentication) find the QR image and save it(it is a temporary image). Then we can run command to get QR code.

Now, we get our QR code, and time to add to pass-otp.

Get our six digital code.

Important: now we need input the code to our github setting or login site to make sure the code stores at the same side, otherwise it will not make effect.

After that, we can use our computer to generate correct code to identify and login our github account.
