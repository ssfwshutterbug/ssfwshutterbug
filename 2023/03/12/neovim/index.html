NEOVIM

I think the most interesting thing about NEOVIM is that it natively supports LSP. So what is LSP?

LSP stands for language server protocol, which first be used in vscode by MicroSoft,it is a standard protocol that our editor and language compiler should follow in order to make our users code much more easily with syntax hint, error hint, syntax autocomplete and so on.

Rust Analyzer

For example, how can i get neovim to support rust language autocomplete? There are two steps:

  1. Download rust-analyzer, which gives us the ability to autocomplete.
  2. Configure neovim to find and use it.

That’s it, only two steps, no need for other plugins, cause neovim natively supports LSP. But in order to make the configuration much more easier, there is a plugin called nvim-lspconfig. And there is also a plugin called maeson which can fully auto configure all. So you can choose one of them, or none and do it manually. The way i choose is nvim-lspconfig.

  • Download rust-analyzer

    There are many ways for us to download rust-analyzer.

    • Download the binary manually and put it to system PATH.
    • Use cargo command to download.
    • Use maeson to download.

    I prefer the manually way.

Lua Language server

Now let’s install and configure lua language server for lua autocomplete.

  • Download lua language server binary

  • Extra lua lauguage server to specified location

  • Configure neovim to use lua language server

    Note:

    The last cmd line will launch lua-language-server when we edit a lua file.

    The file location should be absolute path and cannot be a soft link.

    In a fact, we don’t need to write the last cmd line to manually run lua-language-server, generally nvim-lspconfig will do the job for us if it can find the executable in path. In this case, i didn’t put lua-language-server in environment $PATH, so i need to manually run it with the last cmd line.

Golang Language Server

I use neovim and it support Lsp natively, so the only thing i need to do is installing gopls the go language serer and configuring in neovim.

It will install gopls in our $GOPATH/bin, in order to let neovim to find the executable file, we can move it to ~/.local/bin/.
