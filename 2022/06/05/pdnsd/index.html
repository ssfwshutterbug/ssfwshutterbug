Pdnsd

Today, i find something new, a tool called pdnsd. It stands for persistent dns daemon. With the tool, we can cache normally visit domain (dns) information in our local computer to reduce the time resolving domain name from remote server.

install

configure

At first we need to know something about the format of configuration file.

  1. # and /* */ means comment.
  2. in {} block, we write our configuration.
  3. there are several block of server, and we only need one, we need comment others.
  4. the rest of the configuration file we don’t need modify and it can work perfect.

we need to modify server block of dial-up

  1. change label to resolvconf
  2. add ip with our dns
  3. change file to our /etc/resolv.conf

at last it looks like this:

And we also need to change our /etc/resolv.conf‘s nameserver to 127.0.0.1.

enable and start service

check whether it works

we need a command called nslookup which belongs to a toolset bind.

now check whether it works

when the server shows 127.0.0.1 that means it works.

now when we visit some websites, our computer can directly go to the stored ip address without resolve it from remote dns server. cheers!
