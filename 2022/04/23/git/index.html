BASIC CONFIGURATION

There are three kind of configuration files:

  • system：write file: /etc/gitconfig
  • global：write file: ~/.gitconfig
  • local：write file: .git/config

Personally, i will use local configuration file, because it will not create file in home directory which makes home a less mess folder.

USE GIT TO MANAGE REPO

No matter what the directory is empty or full of folders or files, we can change directory to it and initialize it to a git repo.

After run this command, there comes a .git directory in current folder, nothing else. Yeah, git don’t track any file or fold, we need to specify which to track.

Once we run this command, git will track them the one we specified. What if some files or folders that we don’t track? We can create a file named .gitignore to note. Each line allows one file or folder.

Every file we added will store in a stage area. When we finish a big change or feature, we can take a snapshot with git commit.

Notice:

We can escape add step, combine add and commit with one command: git commit -a -m "commit msg"

SPECIFY WHICH SSH KEY PAIR SHOULD BE USED

when already have a local git repo

If we have more than one repo in github and need to sync to local, we will get a problem: each repo needs different ssh keys, but usually we run ssh-keygen only create one key. So how can we produce different keys and let local git repo knows which key should be used?

The method is specify the key should be used in git config

Tip: in this way, we can put ssh key pair in local git directory .ssh instead of home directory ~/.ssh. If we do it like this, we need add .ssh directory in .gitignore file, otherwise our key pair will shows in public, and it is a risk.

specify with each host/domainname

If we have different host, this method is fine, but not suitable for github since all repo own one domainname.

specify in command directly

It is convenient for clone a new remote repo, without configuration file.

credential

I am in a work with azure devops platform. And sometimes some users ask for my help of git authorized issues.

There are two ways that a user can be used to authorize:

  • ssh
  • http

with ssh method, i clearly know that we need produce a pub key in local machine, and store the pub key to azure devops platform. After that we can download repository without username and password typing.

however, i really don’t know how the way of http works. I searched online and get nothing. Until one day i see my teamwork to modify the username and password in control pannel, and i finally know that.

There are two ways of http authorization works:

  1. with no configure, then we need to type username and password each time we download
  2. with a credential helper, and this can be split with three methods:
    1. cache: store username and password in memory, with a timeout.
    2. store: store username and password in a file, permanently.
    3. other softwares: in win10, is called manager. in win11, called manger-core

cache and store methods will use the OS’s own ability to encrypt username and password

with configure, there are three kinds of level:

  • system: write in /etc/gitconfig
  • global: write in ~/.gitconfig
  • local: write in repo’s .git/config

the low level can override the above level, for example: if we write a configre in local repo, then user local configure instead of system’s.

and if no low level setting, the above level can be used by whole system.

why we use git bash without configure and it used windows credential in default?

because when we install git bash, there comes a config setting in default
